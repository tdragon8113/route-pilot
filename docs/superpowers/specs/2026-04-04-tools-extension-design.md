# 工具功能扩展设计

## 概述

在 RoutePilot 的工具页面新增三个功能模块，用于验证 VPN 是否生效和网络诊断：
1. 公网 IP 查询 - 显示当前公网 IP 及地理位置信息
2. 路由表查看 - 查看系统路由表，支持过滤
3. 路由追踪 - 追踪到目标的网络路径

## 功能详情

### 1. 公网 IP 查询

**目的**：验证当前流量出口是否符合预期（是否走了 VPN）

**显示信息**：
- IP 地址
- 国家
- 城市
- ISP/运营商

**实现方式**：
- 使用免费 API `http://ip-api.com/json/`
- 无需 API Key，每分钟 45 次免费请求限制
- 返回 JSON 格式，包含国家、城市、ISP 等字段

**交互流程**：
1. 用户点击「查询」按钮
2. 显示加载状态
3. 显示查询结果或错误提示

### 2. 路由表查看

**目的**：查看系统当前路由表，分析路由配置

**显示信息**（每条路由）：
- 目标网络（Destination）
- 网关（Gateway）
- 接口（Interface）
- 标志（Flags）

**过滤功能**：
- 按 IP 地址过滤：输入 IP，只显示包含该 IP 的路由
- 按接口过滤：选择接口名（如 ppp0、utun0、en0），只显示该接口的路由

**实现方式**：
- 使用系统命令 `netstat -rn` 获取路由表
- 解析输出提取路由信息
- 过滤在内存中进行

**交互流程**：
1. 自动加载当前路由表
2. 用户可选择接口或输入 IP 进行过滤
3. 实时更新显示结果

### 3. 路由追踪

**目的**：追踪到目标的网络路径，验证是否经过 VPN 接口

**显示信息**（每跳）：
- 序号
- IP 地址
- 主机名（如果能解析）
- 响应时间

**实现方式**：
- 使用系统命令 `traceroute`（macOS 内置）
- 解析命令输出提取每跳信息
- 支持域名和 IP 地址作为目标

**交互流程**：
1. 用户输入目标域名或 IP
2. 点击「追踪」按钮
3. 实时显示每一跳的结果（traceroute 是渐进式输出）
4. 支持「取消」中断进行中的追踪

## 技术细节

### 公网 IP API 响应示例

```json
{
  "status": "success",
  "country": "United States",
  "countryCode": "US",
  "city": "Los Angeles",
  "isp": "Cloudflare, Inc.",
  "query": "104.16.132.229"
}
```

### traceroute 输出解析

```
traceroute to google.com (142.250.185.78), 64 hops max
 1  192.168.1.1 (192.168.1.1)  1.234 ms  1.123 ms  1.089 ms
 2  10.0.0.1 (10.0.0.1)  5.234 ms  5.123 ms  5.089 ms
 3  * * *
```

解析每行提取：序号、IP、主机名（括号内）、响应时间（取第一个）。

### netstat -rn 输出解析

```
Destination        Gateway            Flags        Netif Expire
default            192.168.1.1        UGScg        en0
10.0.0.0/8         10.8.0.1           UGSc        ppp0
127.0.0.1          127.0.0.1          UH           lo0
```

解析每行提取：目标网络、网关、接口名。忽略标题行。

## 错误处理

- 公网 IP 查询失败：显示「查询失败，请检查网络连接」
- 路由表获取失败：显示「获取路由表失败」
- 路由追踪无响应：显示超时提示
- 无效的目标地址：显示「无效的目标地址」
