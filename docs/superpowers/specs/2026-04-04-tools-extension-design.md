# 工具功能扩展设计

## 概述

在 RoutePilot 的工具页面新增两个功能模块，用于验证 VPN 是否生效：
1. 公网 IP 查询 - 显示当前公网 IP 及地理位置信息
2. 路由追踪 - 追踪到目标的网络路径

## 功能详情

### 1. 公网 IP 查询

**目的**：验证当前流量出口是否符合预期（是否走了 VPN）

**显示信息**：
- IP 地址
- 国家
- 城市
- ISP/运营商

**实现方式**：
- 使用免费 API `http://ip-api.com/json/`
- 无需 API Key，每分钟 45 次免费请求限制
- 返回 JSON 格式，包含国家、城市、ISP 等字段

**交互流程**：
1. 用户点击「查询」按钮
2. 显示加载状态
3. 显示查询结果或错误提示

### 2. 路由追踪

**目的**：追踪到目标的网络路径，验证是否经过 VPN 接口

**显示信息**（每跳）：
- 序号
- IP 地址
- 主机名（如果能解析）
- 响应时间

**实现方式**：
- 使用系统命令 `traceroute`（macOS 内置）
- 解析命令输出提取每跳信息
- 支持域名和 IP 地址作为目标

**交互流程**：
1. 用户输入目标域名或 IP
2. 点击「追踪」按钮
3. 实时显示每一跳的结果（traceroute 是渐进式输出）
4. 支持「取消」中断进行中的追踪

## 页面布局

工具页面从上到下依次为：
1. 公网 IP 查询模块
2. 路由查询模块（已有）
3. 路由追踪模块

每个模块使用统一的卡片样式，与现有 UI 保持一致。

## 技术细节

### 公网 IP API 响应示例

```json
{
  "status": "success",
  "country": "United States",
  "countryCode": "US",
  "city": "Los Angeles",
  "isp": "Cloudflare, Inc.",
  "query": "104.16.132.229"
}
```

### traceroute 输出解析

```
traceroute to google.com (142.250.185.78), 64 hops max
 1  192.168.1.1 (192.168.1.1)  1.234 ms  1.123 ms  1.089 ms
 2  10.0.0.1 (10.0.0.1)  5.234 ms  5.123 ms  5.089 ms
 3  * * *
```

解析每行提取：序号、IP、主机名（括号内）、响应时间（取第一个）。

## 错误处理

- 公网 IP 查询失败：显示「查询失败，请检查网络连接」
- 路由追踪无响应：显示超时提示
- 无效的目标地址：显示「无效的目标地址」
